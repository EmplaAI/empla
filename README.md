## What is empla?

empla creates **digital employees** — not chatbots, not task runners, not RPA bots.

A digital employee has goals, forms strategies, takes initiative, and works continuously toward objectives — just like a human employee would. You give it a role and objectives, and it figures out what to do.

```python
from empla.employees import SalesAE
from empla.employees.config import EmployeeConfig

jordan = SalesAE(EmployeeConfig(
    name="Jordan Chen",
    role="sales_ae",
    email="jordan@company.com"
))

await jordan.start()

# Jordan now autonomously:
# - Checks CRM pipeline via HubSpot
# - Reads and responds to incoming leads
# - Schedules meetings on Google Calendar
# - Adapts strategy when pipeline drops
# - Learns from outcomes over time
# ... all without being told what to do
```

## How it works

### BDI Architecture

Every employee runs a continuous **Belief-Desire-Intention** loop — the same model used to understand human reasoning:

```
PERCEIVE → UPDATE BELIEFS → PLAN → EXECUTE → LEARN → repeat
```

- **Beliefs** — What the employee knows about the world. Updated every cycle via tool calls to CRM, email, calendar.
- **Desires** — Goals with measurable targets. "Pipeline coverage ≥ 3.0x." Evaluated by LLM each cycle.
- **Intentions** — Committed plans with steps. Generated by the LLM, executed via tool calls.

The loop runs continuously. Each cycle: perceive the environment, update beliefs, evaluate goal progress, execute the highest-priority work, and learn from outcomes.

### Four Types of Memory

| Type | Purpose |
|------|---------|
| **Episodic** | Past experiences — "Last time pipeline was low, video outreach got 3x response rate" |
| **Semantic** | Facts and knowledge — company info, contact details, learned domain knowledge |
| **Procedural** | What works and what doesn't — reinforced over time, influences future plans |
| **Working** | Current focus — recent observations, active context for LLM decisions |

### Integrations

| Connector | What the employee can do |
|-----------|--------------------------|
| **HubSpot CRM** | Read pipeline, create deals, manage contacts, search |
| **Google Calendar** | Check schedule, book meetings, find availability |
| **Email** | Send, reply, read inbox, follow up |
| **MCP servers** | Connect any MCP-compatible tool |

Connectors call APIs directly. OAuth tokens are auto-injected at startup.

### Trust & Safety

External data (like emails) can contain adversarial content that tries to manipulate the LLM. empla handles this with a **taint-based trust boundary**:

- Structured data (CRM metrics, calendar events) flows freely — no injection risk
- After reading untrusted content (emails, messages), role-specific restrictions activate
- Destructive operations (bulk delete, admin actions) are always blocked
- Per-integration health monitoring detects degraded tools and adapts strategy

## Where empla is headed

### Shadow Mode & Learning from Humans

New employees will shadow human mentors before working independently — observing email style, meeting approach, decision patterns. Not just *what* the mentor does, but *why*. Continuous feedback from humans refines behavior over time.

### Real Meeting Participation

Employees that join calls with voice and video, present with screen sharing, handle Q&A in real-time, and send follow-up notes — full meeting participants, not just transcribers.

### Natural Interaction

No special interface needed. Email jordan@company.com directly. Mention @jordan in Slack. Assign tasks through your existing tools. Give feedback and watch the employee adapt.

### Teams of Digital Employees

Sales AE closes a deal → CSM takes over with full context. PM coordinates with Sales and CS on a product launch. Shared knowledge means one employee's learning benefits the entire team.

### More Employee Types

Product Managers that analyze usage and prioritize roadmaps. Recruiters that source and screen candidates. Executive Assistants that manage calendars and prepare briefs.

## Architecture

See [ARCHITECTURE.md](ARCHITECTURE.md) for the full system diagram.

```
empla/
├── bdi/              # BDI engine (beliefs, goals, intentions)
├── core/
│   ├── loop/         # Proactive execution loop (7 modules)
│   ├── memory/       # 4 memory systems
│   └── tools/        # ToolRouter, trust boundary, health monitor
├── employees/        # Employee types (SalesAE, CSM)
├── integrations/     # HubSpot, Google Calendar, Email
├── llm/              # LLM providers (Claude, OpenAI, Vertex, Azure)
├── models/           # Database models (PostgreSQL)
├── api/              # FastAPI dashboard API
└── services/         # OAuth, credentials, activity recording
```
